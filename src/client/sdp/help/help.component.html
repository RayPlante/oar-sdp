How to build Advanced Search Queries

The Science Data Portal provides an advanced query building feature for more specifically directed searches to the underlying NIST public data repository records through a web interface. Through the Advanced Search Builder interface, users have the capability of constructing a query and if desired, saving this in their browser cache. To create an Advanced Query, the user must first Click on the + Create Query as shown below.

Selecting the +Create New Field will open up the editor for the query entry boxes in the table. This will open a query construction table that allows users to select a Field Type from a drop down list of field names, and enter the value in the Field Text box.

Select the Field Type from the drop down box , then type in the interested search text in Field Text box. The Query will automatically be populated in the main Search box in the top middle of the page. Multiple fields can be queried by selecting the Action button to add another row, i.e. another field. Similarly, fields may be removed from the query by Selecting the Action remove button to the right of the row.

Learning from the patterns using the Advanced Search Builder, in addition to the Search Query Rules, users may enter their advanced query directly in the Search box. Users may also Save their queries for future use by selecting the Save button below the Query construction table. The user may name their query, save it, and then it will appear in the Queries table on the left side for future selection or modification. The saved query will remain available through the application browser window until the user browser cache is cleared. A Saved Query will show in the top banner with a count.

To execute the Advanced Query, a user should click on the main Search button to see the results. Upon execution the search results page listing will appear.

A user can go directly to their query by selecting on Queries and then selecting the More button to go to the query builder interface. An existing query may be modified, deleted or saved again to change the field value and conduct additional searches.

Advanced Search Rules

These rules govern the syntax for a search typed into the SDP search box.

  1. A query is made up of a series of tokens delimited (in the absense of quotes) by one or more spaces. A pair of quotes (containing other text) delimits a single token.  An unpaired quote is taken a literal quote character that does not affect the delimiting of tokens.
  2. Each token represents a constraint on the query; the format determines the type of query:
    1. single word – this is interpreted as a term to be matched to any word in the record
    2. quoted phrase – the characters between the quotes are treated as a phrase that must appear exactly in the record in order to match
    3. and, AND, or, OR – these literal words correspond to constraint joining operators (see 3.)
    4. key=value – this is interpreted as constraint against a specific field in the record.
      1. The value is either a single word or a quoted phrase
      2. For a record to match, the field given by key must contain the value.
  3. The constraints are joined together as follows:*
    1. All single words and quoted phrase constraints (2.2) are joined together via a space (which means AND in this context) and set as the value to the searchphrase constraint.*
      1. *this is a current limitation of the RMM service
    2. If a joining operator token appears between two other tokens, those constraints are joined by that logical operator.
      1. AND, and – intrepreted as a logical AND of the two constraints
      2. OR, or – interpreted as a logical OR of the two constraints
    3. Two non-joining operator tokens next to each other are implicitly joined by a logical OR.

Examples

Search Criteria Keywords Description

Individual terms

NIST Chemistry

Records that include either “NIST” OR “Chemistry” are returned.

Phrase

“NIST Chemistry”

Records that contain the phrase “NIST Chemistry” (the two words adjacent to each other in that order) will be returned. 

Phrase with Individual terms

“NIST Chemistry” Physics

Search Results will match the Phrase “NIST Chemistry” AND (“Physics” OR “NIST” OR “Chemistry”); That is, returned records will contain the phrase “NIST Chemistry”; they may contain “Physics”.

Key Value pair

ContactPoint.fn = “Peter Linstrom”

Records whose field, ContactPoint.fn, contains “Peter Linstrom” are returned.
